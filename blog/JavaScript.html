Oct. 26: JavaScript, Not Just a Pretty Object


Ruby Hashes vs. JavaScript Objects



This week's new focus, JavaScript, was less of a mountain to climb simply because the language has so many similarities with Ruby. While the two languages remain distinct, they still use a number of tools which, despite different naming conventions, accomplish more or less the same role. JavaScript's Objects has functionality that can be easily compared to Ruby's Hash.
A hash, as we know, is a collection of key-value pairs linking a key such as "name" to the value "Charlotte" or "Age" to the number 25.
Objects are JavaScript's way of organizing key-value pairs. The expression of those relationships is, of course, different.
For instance, JavaScript would express that relationship thus:

var woman = {name: "Charlotte", age: "35"}

Ruby will instead say:

woman = {"name" => "Charlotte", "age" => 35}

To access the data, the idea is the same but the methods vary: in JavaScript, you would learn Charlotte's age by calling woman.age; whereas in Ruby, you would write woman[age].
Modifying the content of an object or a hash is equally simple. If I were to add the key "Hometown", I would do this:

woman.hometown = "New York";

in JavaScript, and simply write:

woman['hometown'] = "New York"

in Ruby.
Same ideas, slight differences in syntax and punctuation.
